Supported OS: Windows, Linux

Description

Since Xeneth 2.5 a new device discovery method was introduced to provide a simplified method of device discovery. The user is able to speed up discovery by only enabling the required protocols using the flags argument of the new XCD_EnumerateDevices function. For each discovered device a structure is returned with device information identifying camera name, serial, pid, location and transport. In this sample we will demonstrate a typical device discovery situation and print the device information to console for each discovered device. By making use of the URL it is possible to establish a connection to the device in question.

To continue with establishing a connection refer to "Samples -> C/C++ Samples -> Open a camera and get an image -> Basic Sample" to learn more.

Used functions

The following table lists the functions used in this sample:

Function Description
XCD_EnumerateDevices Perform device enumeration while being able to determine on which protocols device discovery should occur.

Used enumerations

The following table lists the XEnumerationFlags enum and their values used in this sample:

Key Value Description
XEF_Network 0x00000001 Enables the Xenics network protocol.
XEF_Serial 0x00000002 Enables the Xenics serial protocol.
XEF_CameraLink 0x00000004 Enables the CameraLink protocol.
XEF_GigEVision 0x00000008 Enables the GigE Vision protocol.
XEF_CoaXPress 0x00000010 Enables the CoaXPress protocol.
XEF_USB 0x00000020 Enables the USB protocol.
XEF_EnableAll 0x0000FFFF Enable enumeration for all available protocols.
XEF_UseCached 0x01000000 Retrieve devices available in the cache, when this flag is combined with other flags the rest is ignored.
XEF_ReleaseCache 0x02000000 Release devices in the internal cache. It is not neccessary to call XCD_EnumeraDevices using this flag since cleanup is handled automatically on library unload. Still when needed some memory can be released using this flag.

These flags can be OR'ed together to enable one or many protocols. XEF_EnableAll is there for convencience when all protocols should be enabled.
