Supported OS: Windows, Linux

Description

When working with thermally corrected cameras, it is possible to convert the digital values into temperature units. In this sample a connection is made and the thermal calibration pack is loaded. After this the Thermography filter is initialized, "celsius" is provided as an argument to define the unit in which the temperature should be represented. Other possible units are "kelvin" and "fahrenheit".

For better performance when converting the digital values to temperature, a lookup table is prepared prior to the acquisition of images.

Used functions

The following table lists the functions used in this sample:

Function Description
XC_OpenCamera Opens a connection to the camera specified and returns a handle pointing to the camera. The handle is used in all other functions so keep a reference.
XC_CloseCamera Closes and cleans up the connection to the camera specified by the handle which was obtained by calling the XC_OpenCamera function.
XC_IsInitialised Check if the connection was successfully initialized.
XC_StartCapture Start capturing frames.
XC_StopCapture Stop capturing frames.
XC_IsCapturing Check if the device is capturing frames.
XC_GetFrame Used to grab a raw frame from the device.
XC_GetFrameSize Used to determine the native frame size.
XC_LoadCalibration Loads sensor calibration data, and optionally starts the correction if the XLC_StartSoftwareCorrection is specified.
XC_FLT_Queue Queues the filter based on the supplied name into the filter system.
XC_GetMaxValue Retrieve the maximum allowed ADU value.
XC_FLT_ADUToTemperature Convert the supplied ADU value to a temperature unit.
XC_GetWidth Used to determine the width of the camera.
XC_GetHeight Used to determine the height of the camera.
