Supported OS: Windows, Linux

Description

This sample starts from where we left the "Basic Sample" with the difference that instead of capturing just one frame we grab 5 frames in Non-Blocking mode followed by 5 frames in Blocking mode followed by 5 frames in NoConversion & Blocking mode.

The blocking mode is set by setting the flag parameter with one or multiple values of the XGetFrameFlag-enumeration. A table below explains these different flags. Each flag can be passed simultaneously by using the bitwise OR-operator.

Flag Description
XGF_Blocking By default XC_GetFrame will immediately return E_NO_FRAME when no frame is available. If the programmer wants to have the function behave so that it will wait forever until a frame is available then this flag is passed.
XGF_NoConversion Internally for XC_SaveData an 8-bit buffer is updated every frame. When this function is not used, some performance can be gained by setting this flag when grabbing a frame.
XGF_FetchPFF Some types of cameras, like the Onca or Gobi products, support the addition of a footer included with the frame coming from the camera. The footer includes information like timing information, exposure, camera temperature ... The structures that defines this footer data type are located in XFooters.h that is included in this SDK.

Note that in the above table the XGF_FetchPFF flag is also mentioned but its functionality will not be covered in this sample since this is very specific to the type of the camera. Refer to "Samples -> C/C++ Samples -> Open a camera and get an image -> Footers" for more information about footers.

Used functions
