Supported OS: Windows, Linux

Description

To retrieve the footer when using Onca or Gobi based cameras, make sure the frameBuffer is initialized with enough room to hold the frame footer. The function XC_GetFrameFooterLength can be used to calculate the exact buffer size. Once the connection is established, a frame can be grabbed using the GetFrame function, only this time we add the XGF_FetchPFF flag to the GetFrameFlags to signal the API that the frame footer should be included. There are two kinds of footers: software footer, hardware footer. The software footer is created internally in the SDK, the hardware footer is coming straight from the camera. To demonstrate the contents of these footers we print the values to console and clean the connection.

Used functions

The following table lists the functions used in this sample:

Function Description
XC_OpenCamera Opens a connection to the camera specified and returns a handle pointing to the camera. The handle is used in all other functions so keep a reference.
XC_CloseCamera Closes and cleans up the connection to the camera specified by the handle which was obtained by calling the XC_OpenCamera function.
XC_IsInitialised Check if the connection was successfully initialized.
XC_StartCapture Start capturing frames.
XC_StopCapture Stop capturing frames.
XC_IsCapturing Check if the device is capturing frames.
XC_GetFrame Used to grab a raw frame from the device.
XC_GetFrameSize Used to determine the native frame size.
XC_GetFrameFooterLength Retrieve the total size of the hard- and software frame footer.
