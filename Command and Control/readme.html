Supported OS: Windows, Linux

Description

In situations where the image is grabbed by making use of the analog out connection and control of the camera is required, a special attribute is appended to the query part of the connection URL called fg. This allows to inform the SDK which framegrabber will be used for grabbing images. Although, refer to the sample "Using a framegrabber" for the proper usage of these attributes. Below is a table of different values that can be passed:

Values Description
none Start API in command & control mode (no framegrabbing)
XFrameGrabberNative The native framegrabber over Ethernet or USB is used.
XFrameGrabberNationalInstruments for use with the National Instruments framegrabbers like NI PCI-1428 or NI PCIe-1429.
XFrameGrabberEURESYS for use with the Euresys GrabLink series frame grabbers.

In this sample we focus on initializing a connection to the camera in command & control mode. Once the camera is initialised the serial en product id is retrieved, more information on how to use all features of the property system can be found in the Properties-samples.

Used functions

The following table lists the functions used in this sample:

Function Description
XC_OpenCamera Opens a connection to the camera specified and returns a handle pointing to the camera. The handle is used in all other functions so keep a reference.
XC_IsInitialised Check if the connection was successfully initialized.
XC_GetPropertyValueL Gets the property identified by a string of type long.
XC_CloseCamera Closes and cleans up the connection to the camera specified by the handle which was obtained by calling the XC_OpenCamera function.
