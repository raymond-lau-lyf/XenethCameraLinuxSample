Supported OS: Windows, Linux

Description

Sometimes the user is not able to discover the device using the automated discovery procedure, for example GigEVision and the in-house network protocol use UDP broadcasting as the initial discovery packet. It might be that the user is able to ping and connect to this IP address but not able to send and receive the UDP broadcast messages due to network configuration constraints or issues. This was one of the main reasons to introduce an URL that allows the user to use the IP address of the camera when establishing a direct connection. As an example consider the following device list.
