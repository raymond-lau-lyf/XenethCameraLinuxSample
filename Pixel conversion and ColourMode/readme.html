Supported OS: Windows, Linux

Description

This sample starts from where we left the "Basic Sample" with different parameters passed to the XC_GetFrame function to specify another type of frame. Also a few new functions are introduced to show how to use different color modes and how to load an image containing a color profile to apply.

The different frame types that can be requested are listed in the table below:

FrameType Description
FT_NATIVE This is the native frame type of the camera.
FT_8_BPP_GRAY Converts the buffer to 8-bit grayscale.
FT_16_BPP_GRAY Converts the buffer to 16-bit grayscale.
FT_32_BPP_GRAY Converts the buffer to 32-bit grayscale.
FT_32_BPP_RGBA Converts the buffer to a 32-bit ABGR format which can hold color information.
FT_32_BPP_RGB Converts the buffer to a 32-bit BGR format which can hold color information.
FT_32_BPP_BGRA Converts the buffer to a 32-bit ARGB format which can hold color information.
FT_32_BPP_BGR Converts the buffer to a 32-bit RGB format which can hold color information.

The different color modes that can be set are listed in the table below:

ColourMode Description
ColourMode_8 Represent intensities only.
ColourMode_16 Pseudo coloring.
ColourMode_Profile Make use of the color profile provided by the XC_LoadColourProfile function.
ColourMode_Invert Invert the color profile.

Used functions

The same functions as in the "Basic Sample" were used with some extra functions as listed in the table below:

Function Description
XC_SetColourMode Sets the current ColourMode.
XC_LoadColourProfile Loads a color profile image from disk, the dimensions should be 1x256.
XC_GetWidth Used to determine the width of the camera.
XC_GetHeight Used to determine the height of the camera.
