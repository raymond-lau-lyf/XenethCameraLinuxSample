Supported OS: Windows, Linux

Description

Some cameras support uploading and downloading of large binary objects. These objects can be a firmware image, calibration data, overlay data, ... A few functions are exposed to handle this transaction. Since the existense of these blobs are camera specific you should refer to the camera properties documentation to see which properties are supported.

Note
This sample was made with a Gobi Bolometer (Rev1) - F020 camera where we will store the calibration pack, in the camera. Also, a calibration pack is unique to each individual camera, before executing this sample place the calibration pack in this sample's directory. The name should be pack.xca or change the filename in function fopen (line 30) to your custom path.

Used functions

The following table lists the functions used in this sample:

Function Description
XC_OpenCamera Opens a connection to the camera specified and returns a handle pointing to the camera. The handle is used in all other functions so keep a reference.
XC_IsInitialised Check if the connection was successfully initialized.
XC_SetPropertyBlob Upload a binary object to the camera.
XC_CloseCamera Closes and cleans up the connection to the camera specified by the handle which was obtained by calling the XC_OpenCamera function.
